# 1. Src & Tgt 预测极差

在分子/蛋白质图编辑任务中，Action（动作类型）预测准确率高，但 Src 和 Tgt（操作节点）预测极差，这是一个非常经典的现象。Action 预测本质上是一个简单的图级别（Graph-level）多分类任务，而 Pointer Network 预测节点是一个**细粒度的节点级别（Node-level）密集预测任务**，难度不在一个数量级。

通过仔细阅读你提供的 `prediction_heads.py` 代码，我发现了一个**致命的架构逻辑缺陷**，这很可能是导致 target 节点预测崩溃的直接原因。此外，还有几个图神经网络（GNN）在节点选择任务中的常见通病。

以下是深度的原因分析及破局方案：

### 致命缺陷：图节点的绝对索引 Embedding
在 `prediction_heads.py` 的 `PointerNetwork` 中，你使用了 `nn.Embedding` 来嵌入 `src` 节点：
```python
# 你的原代码
self.src_emb = nn.Embedding(max_atoms, hidden_dim)
# ...
s_emb = self.src_emb(src_idx_safe)  # [B, H]
```
**为什么这是错的？**
图数据具有**排列不变性（Permutation Invariance）**。在 Graph A 中，索引为 `5` 的节点可能是一个氧原子；在 Graph B 中，索引为 `5` 的节点可能是一个碳原子。
你用 `nn.Embedding(max_atoms)` 去学习“第 5 个节点”的通用表示，这在化学上是**毫无意义的**。模型根本无法通过一个绝对的数字索引学到任何化学上下文，导致预测 `tgt` 时，由于传入的 `s_emb` 是噪声，`tgt` 的预测必然崩溃。

#### 🛠️ 修复方案 1：使用真实的 GNN 节点特征
在预测 `tgt` 时，应该提取 `src` 节点经过 GNN 编码后的**真实化学特征**，而不是它的索引。

**修改 `prediction_heads.py` 中的 `PointerNetwork`：**
```python
    def __init__(self, ...):
        # ...
        self.action_emb = nn.Embedding(num_actions, hidden_dim)
        # ❌ 删掉这行：self.src_emb = nn.Embedding(max_atoms, hidden_dim)
        self.node_proj  = nn.Linear(node_dim, hidden_dim)
        # ...

    def forward(self, ...):
        # ... 前面预测 src 的代码不变 ...

        if target_src_idx is not None:
            src_idx_safe = target_src_idx.clamp(0, self.max_atoms - 1)
        else:
            src_idx_safe = src_logits.argmax(dim=-1).clamp(0, self.max_atoms - 1)

        # ✅ 核心修复：提取 src 节点真实的 GNN 特征
        B = decoder_state.size(0)
        batch_idx = torch.arange(B, device=dense_nodes.device)
        # dense_nodes: [B, max_nodes, node_dim]
        # 提取选中的 src 节点的原始特征 -> [B, node_dim]
        src_node_feat = dense_nodes[batch_idx, src_idx_safe] 
        
        # 投影到 hidden_dim
        s_emb = self.node_proj(src_node_feat)  # [B, H]

        q_tgt = self.q_tgt(
            torch.cat([decoder_state, a_emb, s_emb], dim=-1)
        ).unsqueeze(1)
        
        # ... 后续代码不变 ...
```
这个修改能让模型在预测 `tgt` 时，真正“看到”刚刚选中的 `src` 原子的化学环境，从而大幅提升 `tgt` 的准确率。

---

### 痛点二：GNN 节点特征的“不可区分性” (Indistinguishability)
如果修复了上述 Bug 后，`src` 和 `tgt` 的预测依然不理想，那么问题大概率出在 **Encoder（GNN）** 上。
在分子图中，经常存在**对称结构**（例如苯环上的 6 个碳原子，或者两个对称的侧链）。普通的 GNN（如 GCN/GIN）在没有额外特征的情况下，会给对称节点输出**完全相同**的 Embedding。
如果两个节点的 Embedding 一模一样，Pointer Network 的 Attention 点积结果也会一模一样，模型只能随机瞎猜，导致 Loss 降不下去。

#### 🛠️ 修复方案 2：引入结构/位置编码 (Structural/Positional Encoding)
你必须在输入 GNN 之前，给节点特征 `x` 注入能够打破对称性的特征：
1.  **Laplacian PE (拉普拉斯位置编码)**：计算图拉普拉斯矩阵的特征向量，作为额外特征拼接到节点上。
2.  **Degree / Centrality**：将节点的度数、中心性（Closeness/Betweenness）作为 One-hot 向量加入。
3.  **3D 坐标 (如果有)**：如果有分子的 3D 构象，引入距离矩阵或相对空间编码。

---

### 痛点三：多任务 Loss 的梯度竞争 (Gradient Domination)
在 `actor_pretrainer.py` 中，你计算了四个 Loss（Action, Src, Tgt, Label）。
Action 是 7 分类（极简单），Src/Tgt 是 $N$ 分类（极难，且充满 Padding）。如果直接相加，Action 的梯度会主导早期训练，模型会“偷懒”只学 Action，放弃学习 Pointer。

#### 🛠️ 修复方案 3：调整 Loss 权重与 Focal Loss
1.  **修改 `loss_strategy.py`（虽然没看到代码，但你应该有这个模块）**：
    给 `src` 和 `tgt` 的 Loss 赋予更高的权重。例如：
    $$ Loss_{total} = 1.0 \times Loss_{action} + 5.0 \times Loss_{src} + 5.0 \times Loss_{tgt} + 1.0 \times Loss_{label} $$
2.  **使用 Focal Loss 替代 CrossEntropy**：
    Pointer Network 预测节点时，存在大量的“简单负样本”（比如那些 Padding 节点，或者明显不相关的节点）。使用 Focal Loss 可以强迫模型关注那些“难以区分的相似节点”。

---

### 痛点四：Teacher Forcing 带来的 Exposure Bias
在训练时，`tgt` 的预测依赖于真实的 `target_src_idx`（Teacher Forcing）。但在推理时，如果 `src` 预测错了，传入的 `s_emb` 就是错的，导致 `tgt` 跟着错（一步错步步错）。

#### 🛠️ 修复方案 4：Scheduled Sampling (计划采样)
在 `actor_network.py` 的 `forward` 中，不要永远 100% 使用 `tf.src`。
可以引入一个概率 $p$（随着 Epoch 逐渐降低）：
*   有 $p$ 的概率使用真实的 `tf.src`。
*   有 $1-p$ 的概率，使用模型自己 `src_logits.argmax()` 预测出的 `src_idx` 去提取 `s_emb`，然后再预测 `tgt`。
这能让模型提前适应推理时的错误，增强鲁棒性。

### 总结执行顺序建议：
1.  **立即执行**：修复 `PointerNetwork` 中 `src_emb` 的绝对索引 Bug（方案 1），这是代码层面的硬伤。
2.  **检查数据**：观察验证集的预测结果，看看模型是不是在对称节点上预测错误。如果是，在数据预处理阶段加入 Laplacian PE。
3.  **调整权重**：在你的 `TRAIN_CFG` 中，把 `src` 和 `tgt` 的 loss 权重调大。
